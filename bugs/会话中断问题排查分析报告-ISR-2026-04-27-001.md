## 三、根因分析

### 3.1 问题 1: 36分钟空白期

Agent 在打开网页后应执行下一步操作（读取页面内容、分析、回复），但日志显示无任何活动。

**推断**: LLM API 请求卡住，未返回响应。

**可能原因**:

1. **dashscope API 服务端延迟**
   - GLM-5 处理复杂多轮对话时可能存在排队或限流
   - 服务端响应超时但未正确返回错误

2. **网络连接问题**
   - 到 `https://coding.dashscope.aliyuncs.com/v1` 的连接不稳定
   - 连接建立后数据传输中断

3. **API 调用超时处理缺陷**
   - OpenAI SDK 10 分钟超时未触发
   - SSE 流已建立，但无数据传输，SDK 未检测到超时

### 3.2 问题 2: 最终 abort

**推断**: nginx `proxy_read_timeout` 触发连接关闭。

**触发流程**:

```
[LLM API 请求挂起，无数据返回]
     ↓
[等待超过 60 秒]
     ↓
[nginx 认为后端超时，关闭 SSE 连接]
     ↓
[后端收到连接关闭信号]
     ↓
[AbortController.signal 被触发]
     ↓
[OpenAI SDK 抛出 APIUserAbortError]
     ↓
[AgentService 记录 stopReason: "aborted", errorMessage: "Request was aborted."]
```

**验证依据**:

- 错误信息 `"Request was aborted."` 来自 OpenAI SDK `APIUserAbortError`
- nginx 无 `proxy_read_timeout` 配置，默认 60 秒超时
- SSE 流在等待期间无心跳事件，无法保持连接

## 四、修复建议

### 4.1 优先级 P0: nginx 超时配置

**文件**: `/app/deploy/nginx/default.conf`

**修改建议**:

```nginx
location / {
    proxy_pass http://ugk-pi:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection $connection_upgrade;

    # 新增: SSE 流式响应超时配置
    proxy_read_timeout 600s;   # 10 分钟，与 OpenAI SDK timeout 一致
    proxy_send_timeout 600s;   # 10 分钟

    # 已有配置，保持
    proxy_buffering off;
}
```

### 4.2 优先级 P1: SSE 心跳机制

**目标**: 在等待 LLM API 响应期间保持 SSE 连接活跃。

**实现位置**: `/app/src/agent/agent-service.ts` 或 `/app/src/agent/agent-run-events.ts`

**建议代码**:

```typescript
// 在 AgentService.runChat() 中添加心跳定时器
const heartbeatInterval = setInterval(() => {
  if (activeRun && !activeRun.interrupted && state.loading) {
    emitBufferedRunEvent({
      type: "heartbeat",
      timestamp: Date.now(),
    });
  }
}, 30000); // 每 30 秒发送一次心跳

// 在 runChat finally 中清理
clearInterval(heartbeatInterval);
```

### 4.3 优先级 P2: 前端超时检测

**文件**: `/app/src/ui/playground-stream-controller.ts`

**修改建议**: 在 `readEventStream()` 函数中添加超时检测：

```typescript
const READ_TIMEOUT_MS = 600000; // 10 分钟
let lastDataTime = Date.now();

const timeoutCheck = setInterval(() => {
  if (Date.now() - lastDataTime > READ_TIMEOUT_MS) {
    showError("流式响应超时，请刷新页面重试");
    reader.cancel();
    clearInterval(timeoutCheck);
  }
}, 30000);

// 每次收到数据时更新时间戳
const { value, done } = await reader.read();
lastDataTime = Date.now();
```
