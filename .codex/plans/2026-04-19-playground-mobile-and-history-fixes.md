# 2026-04-19 Playground 修复与移动端适配计划

## 背景

用户要求按以下顺序完成 `playground` 前端修复与优化：

1. 修复技能结构代码块在消息里显示成占位符的问题
2. 修复点击“新会话”后当前页面历史被直接清空的问题
3. 修复消息发送后输入框没有立即清空的问题
4. 在不影响现有桌面端显示的前提下，为手机 Web 做一套沿用当前视觉风格的专属界面

本次任务已获得执行授权，直接进入实现。

## 真实入口

- 页面结构、样式、脚本：`src/ui/playground.ts`
- 页面回归断言：`test/server.test.ts`
- 当前前端口径文档：`docs/playground-current.md`
- 统一更新记录：`docs/change-log.md`

## 方案决策

### 1. Markdown 代码块渲染

- 根因不是后端，而是 `renderPlaygroundMarkdown()` 的代码块占位符在“普通段落 + 紧接代码块”场景下没有被切成独立 block。
- 修复策略：让 fenced code block 在 token 化时天然形成独立段落，避免再把 `CODEBLOCK0` 当普通文本渲染。

### 2. 新会话后的历史保留

- 不能把旧会话消息直接混进新会话的持久化数据，否则会污染按 `conversationId` 分离的本地历史。
- 修复策略：新增“当前页历史归档区”。
  - 点击“新会话”前，先把当前 transcript DOM 归档到页面顶部的历史区
  - 当前会话状态与本地持久化仍按新的 `conversationId` 独立管理
  - 这样既不污染新会话数据，也不会让用户觉得点一下按钮历史就蒸发了

### 3. 发送后立即清空输入框

- 用户需要即时反馈，不能等到 `done` 事件才清空输入框。
- 修复策略：发送/排队请求发起后立即清空 composer，并保留一份草稿快照；若请求失败，再把草稿恢复回输入区。

### 4. 手机 Web 专属界面

- 不改桌面端主视觉，只在移动端断点内收口布局。
- 修复策略：
  - 桌面端保留现有右侧操作区
  - 手机端隐藏右侧操作区，改为顶部菜单入口
  - 手机端单独调整顶部、英雄区、transcript、composer、底部操作按钮和归档区间距
