认证机制分析

🔐
必需的认证 Headers
Header 来源 获取难度 状态
authorization 固定 Bearer Token 公开已知 ✓ 可获取
x-csrf-token ct0 Cookie 登录后自动生成 ✓ 可获取
auth_token 登录 Cookie 登录后获取 ✓ 可获取
x-client-transaction-id 浏览器动态生成 反爬保护 ✗ 无法复制
⚠
关键障碍：x-client-transaction-id

这是 X 从 2023 年开始启用的反爬机制：

• 由浏览器内部 JS 动态计算生成
• 与浏览器指纹、时间戳、请求参数绑定
• 每次请求值都不同
• 无法在容器或外部环境中伪造有效值

调研结论

X/Twitter 不存在可以"简单借用登录态"直接调用的数据接口。

由于 x-client-transaction-id 反爬保护机制的存在，任何绕过浏览器的直接 API 调用都会返回 401 错误。这不是 API 设计问题，而是 X 故意的反爬策略。

✓

唯一可行路径

通过 CDP (Chrome DevTools Protocol) 控制真实浏览器，在页面内执行 JavaScript 提取已渲染的 DOM 数据。这是目前获取 X 搜索数据唯一稳定可靠的方式。
